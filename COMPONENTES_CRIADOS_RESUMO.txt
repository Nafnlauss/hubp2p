TABELA DE BANCO NECESSÁRIA
================================================================================

CREATE TABLE user_profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id),
  email VARCHAR NOT NULL UNIQUE,
  full_name VARCHAR NOT NULL,
  cpf VARCHAR(11) NOT NULL UNIQUE,
  phone VARCHAR(11) NOT NULL,
  is_admin BOOLEAN DEFAULT FALSE,
  is_verified BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

COMO COMEÇAR
================================================================================

1. IMPORTAR NAVBAR E FOOTER NO LAYOUT:

   import { Navbar } from '@/components/navbar';
   import { Footer } from '@/components/footer';

   export default function RootLayout({ children }) {
     return (
       <html>
         <body>
           <Navbar />
           <main className="pt-16">{children}</main>
           <Footer />
         </body>
       </html>
     );
   }

2. USAR EM COMPONENTES:

   'use client';

   import { useUser, useRequireAuth } from '@/hooks/use-user';
   import { StatusBadge } from '@/components/status-badge';
   import { formatCurrency } from '@/lib/utils/format';

   export function MyComponent() {
     useRequireAuth('/login'); // Protege a rota
     const { user, profile } = useUser();

     return <div>{formatCurrency(1500.5)}</div>;
   }

================================================================================
SUPORTE E TROUBLESHOOTING
================================================================================

❌ Navbar não aparece
   → Use 'use client' no layout
   → Adicione pt-16 no main

❌ useUser retorna null
   → Verifique se user está autenticado
   → Verifique se tabela user_profiles existe

❌ TransactionTimer não atualiza
   → Use 'use client'
   → Passe Date ou ISO string válida

❌ Validações não funcionam
   → Verifique import: @/lib/utils/format
   → Alguns CPFs/Phones são inválidos propositalmente
